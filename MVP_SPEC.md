# MVP Feature Specification

## 1. MVP 목표

가장 먼저 검증할 것은 아래 3가지입니다.

- 사용자가 회원가입 후 실제로 채팅을 시작할 수 있는가
- `일반 채팅`과 `랜덤 채팅`의 차이를 명확하게 체감하는가
- 익명 채팅에서도 최소한의 안전 장치가 동작하는가

## 2. MVP 범위

### 포함 기능

1. 회원가입
2. 로그인 / 로그아웃
3. 내 프로필 조회 및 수정
4. 친구 요청 / 수락 / 거절
5. 일반 1:1 채팅
6. 랜덤 1:1 채팅 매칭
7. 랜덤 채팅 익명 닉네임 자동 생성
8. 메시지 송수신
9. 읽음 처리
10. 사용자 차단
11. 사용자 신고
12. 기본 테마 선택 / 적용

### 제외 기능

1. 일반 그룹 채팅
2. 랜덤 2:2 / 3인 / 4인 매칭
3. 음성 통화
4. 파일 전송
5. 선물하기
6. 테마 공유 마켓
7. 관리자 운영 대시보드의 고도화 기능

## 3. 사용자 유형

### 일반 사용자
- 회원가입 및 로그인 가능
- 친구 추가 가능
- 일반 채팅과 랜덤 채팅 사용 가능
- 신고 / 차단 가능
- 테마 선택 가능

### 운영자
- 신고 내역 검토 가능
- 제재 대상 확인 가능

## 4. 기능 명세

### 4.1 회원가입

조건:

- 이메일, 비밀번호, 표시 이름 입력
- 이메일 중복 불가
- 비밀번호는 해시 저장

완료 조건:

- 가입 완료 후 로그인 가능

### 4.2 로그인

조건:

- 이메일과 비밀번호 검증
- 정지 계정은 로그인 제한

완료 조건:

- 토큰 또는 세션 발급

### 4.3 친구 요청

조건:

- 사용자 검색 또는 프로필 진입 후 요청 가능
- 이미 친구이거나 대기 중이면 중복 요청 불가
- 차단 관계가 있으면 요청 불가

완료 조건:

- 상대가 수락하면 일반 채팅 생성 가능

### 4.4 일반 1:1 채팅

조건:

- 친구 관계일 때만 시작 가능
- 실명 대신 프로필 표시 이름 사용
- 텍스트 메시지 송수신 가능
- 읽음 상태 확인 가능

제한:

- MVP에서는 파일, 선물, 통화 비활성화

### 4.5 랜덤 1:1 채팅

조건:

- 사용자가 랜덤 채팅 시작 버튼 클릭
- 매칭 대기열 등록
- 다른 대기 사용자와 1:1 매칭
- 입장 시 자동 익명 닉네임 부여

제한:

- 프로필, 실제 이름, 사용자 아이디 비공개
- 파일 전송 금지
- 선물 전송 금지
- 통화 기능 없음

종료:

- 채팅 종료 후 재매칭 가능

### 4.6 신고

조건:

- 채팅방 또는 특정 메시지 기준 신고 가능
- 사유 카테고리 선택 가능

완료 조건:

- 운영자가 검토할 수 있도록 저장

### 4.7 차단

조건:

- 상대 사용자 차단 가능

효과:

- 일반 채팅 시작 차단
- 친구 요청 차단
- 랜덤 매칭에서 재노출 최소화 로직 적용 가능

### 4.8 테마 적용

조건:

- 시스템 기본 테마 목록 제공
- 사용자가 1개 선택 가능

완료 조건:

- 앱 주요 컬러와 채팅 UI에 반영

## 5. 핵심 사용자 흐름

### 흐름 A: 일반 채팅

1. 회원가입
2. 로그인
3. 사용자 검색
4. 친구 요청
5. 상대 수락
6. 일반 채팅방 생성
7. 메시지 송수신

### 흐름 B: 랜덤 채팅

1. 로그인
2. 홈에서 랜덤 채팅 선택
3. 매칭 대기
4. 익명 닉네임 배정
5. 랜덤 1:1 채팅 시작
6. 신고 또는 종료 가능

## 6. 비기능 요구사항

### 보안
- 비밀번호 해시 저장
- 인증 토큰 검증
- 랜덤 채팅 개인정보 비노출

### 실시간성
- 메시지 지연 최소화
- 읽음 상태 실시간 반영 가능 구조

### 운영성
- 신고 내역 저장
- 차단 내역 저장
- 최소 로그 추적 가능

## 7. 성공 기준

- 사용자가 가입 후 3분 안에 첫 채팅까지 도달 가능
- 일반 채팅과 랜덤 채팅의 권한 차이가 명확함
- 신고/차단이 실제로 채팅 경험에 반영됨
- 테마 변경이 UI에 즉시 반영됨
