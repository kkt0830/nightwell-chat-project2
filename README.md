# Chat Service Planning Pack

`project2`에는 아래 4개 산출물을 정리했습니다.

- `DB_DESIGN.md`: 서비스용 DB 테이블 설계
- `MVP_SPEC.md`: MVP 기능 명세서
- `SCREEN_STRUCTURE_AND_WIREFRAMES.md`: 화면 구조와 와이어프레임
- `index.html`: 위 내용을 빠르게 훑어볼 수 있는 요약 페이지

서비스 핵심 방향은 아래와 같습니다.

- 로그인 이후 `일반 채팅`과 `랜덤 채팅`으로 진입
- `일반 채팅`은 친구 기반 대화, 파일, 통화, 선물 등 확장 가능
- `랜덤 채팅`은 익명 기반 매칭형 대화, 기능 제한 강제
- 테마는 사용자 제작 및 적용 가능

MVP 범위는 과도한 확장을 막기 위해 아래에 집중했습니다.

- 회원가입 / 로그인
- 친구 추가
- 일반 1:1 채팅
- 랜덤 1:1 채팅
- 신고 / 차단
- 기본 테마 선택 및 적용
